## 🟢 Minor Issues

### N-NEW-1 (carry-over) — Client timer missing rarity speed modifier
**File:** `MainClient.client.lua` (timer loop)  
Client uses `base / speed`. Server uses `base × rarityMod / speed`. Mythic = 48-min visual drift.

```lua
-- Fix:
local RARITY_TIME_MODS = {Common=1.00,Uncommon=1.00,Rare=0.95,Epic=0.90,Legendary=0.85,Mythic=0.80}
local rarityMod = (seedDef and RARITY_TIME_MODS[seedDef.rarity]) or 1.0
local effectiveTime = math.ceil(seedDef.harvestTime * rarityMod / math.max(harvestSpeed, 0.1))
```

### N-NEW-2 (carry-over) — UpgradePanel CloseBtn not wired
**File:** `MainClient.client.lua`  
`BuildGUI.lua` creates CloseBtn inside UpgradePanel — never connected.

```lua
local closeBtn = UpgradePanel:FindFirstChild("CloseBtn") :: TextButton?
if closeBtn then closeBtn.Activated:Connect(function() UpgradePanel.Visible = false end) end
```

### N-NEW-3 (carry-over) — SeedData.Get() unguarded in client timer
**File:** `MainClient.client.lua` (timer loop)  
Uses `assert()` — bad seedId crashes entire timer coroutine silently.

```lua
local ok, seedDef = pcall(SeedData.Get, state.seedId :: string)
```

### N-NEW-4 (new) — Plant/Harvest/Upgrade handlers have no rate limit
**File:** `GameManager.server.lua`  
5 handlers missing per-player cooldown. Low risk, defensive hardening.

```lua
local actionCooldowns: {[number]: number} = {}
local ACTION_COOLDOWN = 0.3
```
